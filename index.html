Split-and-Conquer: Client-Decoupled Split Federated Learning in Heterogeneous Environments

Anonymous Authors
Paper Code

Overview

Split-and-Conquer is a Split Federated Learning (SFL) framework designed for heterogeneous environments. It removes strict client–server backpropagation dependency so clients can personalize locally and the server can train robustly under stragglers.

  • Client personalization: client-side layers are fully personalized, only a lightweight shared head is shared.
  • Client decoupling: clients compute loss and update client-side layers without waiting for server gradients.
  • Straggler robustness: server trains efficiently using a replay memory of client embeddings.

Motivation

Existing SFL methods suffer from two practical bottlenecks in real deployments:

  • Weak personalization: federated aggregation on client-side layers enforces shared representations, hurting performance under non-IID data and heterogeneous devices.
  • Straggler delays: clients must wait for server-side backpropagation, so slow clients delay end-to-end progress.

The core question: Can we redesign SFL to enable client-specific personalization and straggler-robust training efficiency at the same time?

Method: Split-and-Conquer

We split training into two parallel processes: (1) client-side personalization (fully local updates) and (2) server-side representation learning (updates using replay memory). The only federated component is a lightweight shared head.

What’s new?

  • Compatibility-aware client update: before training the full local model, each client first updates only the personalized layers while freezing the aggregated shared head.
  • Server training with replay memory: clients send latent embeddings; the server stores them in a replay memory and trains a large server-side model independent of client availability.
  • Decoupled critical path: removing server-to-client backpropagation lets client updates and server updates run in parallel.

Results on SST-5 dataset with Llama-3.2-3B model

Split-and-Conquer delivers better accuracy under heterogeneity while improving end-to-end training efficiency under stragglers.

Robust personalization across diverse data heterogeneity

In highly heterogeneous environments (Dirichlet α=0.05), Split-and-Conquer achieves 46.5% accuracy, substantially surpassing existing methods such as MU-SplitFed (32.6%) and DC-SFL (29.6%). Similar gains are observed at α=0.1, where our method reaches 47.7% accuracy, outperforming the best baseline by over 11 percentage points.

Even in homogeneous settings (α=100), Split-and-Conquer achieves the highest accuracy of 58.6%, demonstrating its ability to improve performance without sacrificing global representation learning. These results confirm that decoupling client personalization from server-side training enables more effective and robust learning across diverse client data distributions.

Straggler-robust training efficiency

As the straggler ratio increases, conventional split federated learning methods suffer from longer round times and degraded training efficiency due to their reliance on tightly synchronized client–server backpropagation. Because client updates depend on server-side gradient computation, slow or delayed clients directly block overall training progress, causing inefficiencies to accumulate as system heterogeneity grows. Split-and-Conquer addresses this limitation by decoupling client-side personalization from server-side representation learning, allowing clients to update their local models independently while the server asynchronously learns from client embeddings. This decoupled design removes slow clients from the critical training path, enabling stable round times and consistently higher normalized training-time efficiency. As a result, Split-and-Conquer provides significantly more robust and scalable training performance in heterogeneous and straggler-prone federated environments.

Ablations: why it works

We ablate three components to show what drives performance and stability: server capacity, replay memory size, and the compatibility step (freezing the shared head) that prevents mismatch during client personalization.

  • Server capacity: larger server models improve representation learning
  • Replay memory: more replay stabilizes learning under partial participation
  • Compatibility step: freezing the aggregated head improves robustness
